Flow without Saga

in the previous section, we have built the basic project with happy scenario (success-scenario). this flow will explain that process how it was executed.


Flow with Saga (With compensation transactions)

now we are going to add the compensation for all the process (command-process / command execution) which executed on that flow. You can see here each command execution has a compensation if the main execution failed in some case. that means just think the request goes through the first 3 service successfully, but in the increment points will be failed. then what to do? then we have to decide whether the process we can skip and continue to forward. or we have ro revert all the execution that we have been doing. most of the time we have to revert all the process we have done so far. that why the saga has compensation transaction for all the executions (for command execution).



Identifying the compensations transactions

for the convenience, let's use a table.
service execution/action/event process-transaction compensation-transaction
order-service COMMAND ORDER_CREATED ORDER_CANCELED
user-service QUERY CHECK_USER ----
payment-service COMMAND MAKE_PAYMENT REFUND_PAYMENT
user-service COMMAND INCREMENT_POINTS DECREMENT_POINTS
delivery-service COMMAND ADD_TO_DELIVERY CANCEL_DELIVERY


Creating compensation for create-order

We already have the success scenario (create-order) without the compensation. due to the create-order is a command execution, th execution should have a compensation. So, we are going to create the next method of compensation for creating the order in the order-service. (cancel-order). this is an internal service's method. because, the invocation will be done in its own service. (without accessing another service)

# service execution/action/event process-transaction compensation-transaction
1 order-service COMMAND ORDER_CREATED ORDER_CANCELED
2 user-service QUERY CHECK_USER ----
3 payment-service COMMAND MAKE_PAYMENT REFUND_PAYMENT
4 user-service COMMAND INCREMENT_POINTS DECREMENT_POINTS
5 delivery-service COMMAND ADD_TO_DELIVERY CANCEL_DELIVERY
Updated One
order-service > src/main/java/org/mono/stacksaga/example/orderservice/service/internal/InternalCustomerOrderService.java

The newly added method as the compensation of the create-order is public void createOrderRevert(String reason, String orderUid){}



**The 2nd execution is checking the user activeness. but it is query execution. so it shouldn't have a compensation transaction.
