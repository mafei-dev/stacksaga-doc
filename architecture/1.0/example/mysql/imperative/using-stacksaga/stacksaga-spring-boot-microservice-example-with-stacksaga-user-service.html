user-service - migration to SAGA

In one of past few sessions we have done our user-service without moving to the Saga design pattern. now we are going to use that service to migrate to Saga design pattern by using StackSaga. now you know that when the order-service migrates to the StackSaga in the previous session, one additional method was created apart from the default method to invoke the compensation-transaction. in that method called an API from the user-service. that's what we are going to implement here.

Note

Adding StackSaga dependencies is not required in this application. but, in terms of Real-time circuit breaker pattern, it would be better if you add StackSaga dependencies in to each services. if you don't add the framework into the service, Real-time circuit breaker is not used. to get full features of the framework, you can use the dependencies in each application. to see how it works please refer this.
